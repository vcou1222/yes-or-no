# 《Yes or No》Web MVP 需求文档

## 1. 项目概述

### 1.1 项目名称

**Yes or No**

### 1.2 产品定位

一个极简、好看、有情绪感的决策外包 Web。

用户输入自己纠结的问题，系统用预设规则给出一个明确决定，并配上一句有共鸣的理由，帮助用户结束犹豫。

### 1.3 核心一句话

> 写下你纠结的事情，我替你做这个决定。

### 1.4 MVP 目标

本版本不接入 AI，不做复杂逻辑，优先验证：

1. 用户是否愿意输入真实问题；
2. 用户是否愿意接受系统给出的决定；
3. 用户是否愿意截图或分享结果；
4. 小红书用户看到后是否会点击体验。

---

# 2. 目标用户

## 2.1 核心用户

20–35 岁城市年轻人，尤其是处在工作、生活、情感、人生方向焦虑中的人。

他们通常有以下状态：

* 想很多，但很难行动；
* 经常在“要不要”之间反复横跳；
* 不是真的缺信息，而是缺一个推动自己行动的理由；
* 不一定想听理性分析，更想听一句“你就这样做”。

## 2.2 典型用户问题

```text
我要不要辞职？
我要不要离开现在这座城市？
我要不要继续这段关系？
我要不要开始做副业？
我要不要买这个东西？
我要不要主动联系他？
我要不要继续坚持？
```

## 2.3 用户核心心理

用户不是来找“正确答案”的，而是来找：

> 一个替自己结束犹豫的理由。

# 3. 产品原则

## 3.1 极简

整个产品只保留一个主流程：

```text
输入问题 → 点击按钮 → 得到决定 → 生成卡片
```

MVP 不做：

* 登录注册；
* 用户中心；
* 历史记录；
* 复杂分类；
* 付费功能；
* AI 对话；
* 多页面导航。

## 3.2 不让用户动脑子

用户进入页面后，不需要理解规则，不需要选择模式，不需要填写复杂信息。

页面只问一件事：

```text
你在纠结什么？
```

## 3.3 情绪优先

这个产品不是效率工具，而是情绪产品。

所以文案要像“一个人”在说话，不要像系统提示。

错误示例：

```text
系统正在分析您的问题
```

正确示例：

```text
✦ 我在想一想…… ✦
```

## 3.4 强视觉吸引

UI 要适合小红书传播，截图之后也要好看。

关键词：

* 黑色背景；
* 白色大字；
* 紫色微光；
* 极简布局；
* 仪式感；
* 结果卡片有冲击力。

# 4. 页面结构

本项目只需要一个页面。

页面路径：

```text
/
```

页面分为两个状态：

1. 初始输入状态；
2. 决策结果状态。

结果状态可以直接在同一页向下显示，不需要跳转页面。

# 8. 决策逻辑

MVP 不调用 AI，采用本地预设逻辑。

## 8.1 基础逻辑

用户点击「帮我决定」后：

1. 获取用户输入；
2. 随机生成一个决策类型；
3. 根据类型随机匹配一句理由；
4. 展示结果。

## 8.2 决策类型权重

建议不要完全三等分。

为了让用户更容易产生“被推动”的感觉，可以设置：

```js
const decisionWeights = {
  go: 0.45,
  no: 0.25,
  wait: 0.30
}
```

对应：

```text
go → 去做
no → 不要做
wait → 再等等
```


## 8.3 简单关键词规则

MVP 可加入轻量关键词判断，让结果更“像是懂我”。

### 工作类关键词

```text
辞职、工作、上班、老板、同事、公司、面试、跳槽、offer
```

如果命中工作类关键词，可以优先使用工作类理由。

### 情感类关键词

```text
分手、复合、喜欢、联系、表白、关系、对象、前任、他、她
```

如果命中情感类关键词，可以优先使用关系类理由。

### 行动类关键词

```text
开始、坚持、学习、做、自媒体、副业、创业、报名、尝试
```

如果命中行动类关键词，可以优先使用行动类理由。

### 消费类关键词

```text
买、下单、消费、花钱、贵、便宜、值得、手机、衣服
```

如果命中消费类关键词，可以优先使用消费类理由。

如果没有命中关键词，就使用通用理由库。

## 13.4 决策函数

```ts
function decide(question: string): DecisionResult {
  const category = detectCategory(question);
  const type = weightedRandomDecision();
  const reason = pickReason(type, category);

  return {
    type,
    label: getDecisionLabel(type),
    reason,
  };
}
```

## 13.5 分类检测函数

```ts
function detectCategory(question: string) {
  if (/(辞职|工作|上班|老板|同事|公司|面试|跳槽|offer)/.test(question)) {
    return "work";
  }

  if (/(分手|复合|喜欢|联系|表白|关系|对象|前任|他|她)/.test(question)) {
    return "relationship";
  }

  if (/(开始|坚持|学习|做|副业|创业|报名|尝试)/.test(question)) {
    return "action";
  }

  if (/(买|下单|消费|花钱|贵|便宜|值得|手机|衣服)/.test(question)) {
    return "consume";
  }

  return "general";
}
```
